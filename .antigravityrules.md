## 2. Core Architectural & Database Rules

> [!IMPORTANT]
> **Database Schema Design Rules (Strict Compliance Required)**:
> 1. **No Enums for New Features**: The user prefers **lookup tables** and **foreign keys with proper constraints** instead of DB-level enums.
> 2. **Independent Primary Keys**: Every table must have its own independent, autoincrementing primary key (`id`). Avoid using foreign keys directly as primary keys.
> 3. **Referential Integrity**: Always apply `ondelete="CASCADE"` on foreign keys and `cascade="all, delete-orphan"` on parent relationships in SQLAlchemy to ensure clean, orphan-free deletions.

## 3. Backend Guidelines (FastAPI)

* **Directory**: `backend/`
* **Python Runtime**: `Python 3.12.9`
* **Virtual Environment**: `.venv` in the `backend/` root directory.
* **Test Suite execution**:
  * **CRITICAL**: Do NOT run the global `pytest` command (it triggers system architecture library mismatches on macOS).
  * **Rule**: Always invoke the local virtual environment's test runner directly:
    ```bash
    .venv/bin/pytest
    ```

## 4. Frontend Guidelines (React + Vite + Tailwind)

* **Directory**: `frontend/`
* **Styling Strategy**: Tailwind CSS with custom Vanilla CSS overrides in `index.css`.
* **Dark Mode & Styling Aesthetics**:
  * Use a class-based dark mode (`darkMode: 'class'` in `tailwind.config.ts`).
  * Ensure **Zero Flicker on Reload**: Keep the inline head script in `index.html` intact to parse theme preferences immediately before rendering.
  * Favor highly responsive, glassmorphic, premium visual designs (harmonic custom HSL/RGB palettes, dark indigo/emerald space-theme gradients, smooth micro-animations).

## 6. Git & GitHub Pull Request Workflow

When a feature implementation is complete and verified, Antigravity can automate git and pull request tasks:

1. **Branching**: Checkout a clean feature branch from `main` based on the JIRA issue:
   ```bash
   git checkout -b feature/[JIRA-KEY]-[issue-description]
   ```
2. **Staging & Commits**: Commit code with clean, Conventional Commit messages:
   ```bash
   git add .
   git commit -m "feat([scope]): [JIRA-KEY] [short description of change]"
   ```
3. **Publishing**: Push the feature branch to the remote origin:
   ```bash
   git push origin feature/[JIRA-KEY]-[issue-description]
   ```
4. **Pull Request**: Create a rich Pull Request using the `gh` CLI, automatically populating the PR body with the conversation's `walkthrough.md`:
   ```bash
   gh pr create --title "feat: [JIRA-KEY] [brief title]" --body-file "[walkthrough_path]" --web
   ```
